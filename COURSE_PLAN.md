# Kotlin Masterclass: Course Syllabus and Learning Path

This document details the course curriculum and syllabus. It is designed to take students from knowing nothing about programming to writing advanced concurrent applications with Kotlin.

---

## 📅 Curriculum Breakdown

### 🛠️ Module 1: Fundamentals (Lessons 01 - 05)
* **Goal**: Establish the development environment, learn basic syntax, read and write data to/from the terminal, and declare variables.
* **Lessons**:
  1. **Lesson 01**: Setup & Installation (JDK, Android Studio, Hello World)
  2. **Lesson 02**: Variables (`val` vs `var`, Type Inference, Constants)
  3. **Lesson 03**: Data Types (Numbers, Booleans, Characters, Strings, Type Casting)
  4. **Lesson 04**: Operators (Arithmetic, Comparison, Logical, Assignment)
  5. **Lesson 05**: Input & Output (Reading from console, Formatting outputs)

### 🔀 Module 2: Logic & Control Flow (Lessons 06 - 07)
* **Goal**: Write programs that make decisions and repeat blocks of instructions using loop structures.
* **Lessons**:
  1. **Lesson 06**: Conditional Statements (`if`, `else`, `when` expression)
  2. **Lesson 07**: Loops (`for`, `while`, `do-while`, control flow jumps like `break`/`continue`, pattern building)

### 📦 Module 3: Modular Code with Functions (Lesson 08)
* **Goal**: Modularize calculations, parameters, defaults, and single expression syntax.
* **Lessons**:
  1. **Lesson 08**: Functions (Parameters, Return values, Named arguments, Default arguments, Single-expression functions)

### 🗄️ Module 4: Storing Groups of Data (Lessons 09 - 10)
* **Goal**: Explore static vs dynamic datasets. Learn basic array indexing and structural collection types.
* **Lessons**:
  1. **Lesson 09**: Arrays (Creation, Indexing, Traversing, Basic operations)
  2. **Lesson 10**: Collections (Lists, Sets, Maps, and their Mutable equivalents)

### 🏛️ Module 5: Object-Oriented Programming (Lessons 12)
* **Goal**: Model real-world objects in code using modern class hierarchies, interfaces, and polymorphism.
* **Lessons**:
  1. **Lesson 12**: OOP Core (Classes, Constructors, Encapsulation, Inheritance, Polymorphism, Interfaces, Abstract Classes)
  * *Note: Lesson 11 is Null Safety, which is grouped under Kotlin Specifics.*

### 💎 Module 6: Kotlin-Specific Paradigms (Lessons 11, 13 - 16)
* **Goal**: Leverage Kotlin's safety guarantees and syntactic features that make code concise and readable.
* **Lessons**:
  1. **Lesson 11**: Null Safety (Nullable types, Safe calls, Elvis operator, Smart casting)
  2. **Lesson 13**: Data Classes (Automatic generating of `equals`, `hashCode`, `toString`, `copy`)
  3. **Lesson 14**: Enum Classes (Representing fixed sets of values, properties, and custom methods)
  4. **Lesson 15**: Sealed Classes & Interfaces (Representing restricted class hierarchies for state machines)
  5. **Lesson 16**: Extension Functions (Adding new functionality to existing classes without subclassing)

### ⚡ Module 7: Functional Programming & Lambdas (Lessons 17 - 18)
* **Goal**: Learn to treat functions as first-class citizens, pass closures, and transform data pipelines.
* **Lessons**:
  1. **Lesson 17**: Lambdas & Anonymous Functions (Closures, the implicit `it` parameter)
  2. **Lesson 18**: Higher-Order Functions (Functions that receive or return other functions, and standard operations like `map`, `filter`, `reduce`, `fold`)

### 🚀 Module 8: Advanced Kotlin (Lessons 19 - 22)
* **Goal**: Master compiler type-safety with generics, handle exceptions, manipulate the file system, and run code asynchronously.
* **Lessons**:
  1. **Lesson 19**: Generics (Type parameters, Generic classes, and Generic constraints)
  2. **Lesson 20**: Exception Handling (`try`, `catch`, `finally`, custom exception types)
  3. **Lesson 21**: File Handling (Reading/writing files on the JVM)
  4. **Lesson 22**: Coroutines (Asynchronous code, Coroutine scopes, dispatchers, launch vs async, delay)

---

## 🎓 Project Milestones

Throughout the course, students apply what they've learned in practical assignments:

1. **Milestone 1**: Arithmetic Console Calculator (After Module 3)
2. **Milestone 2**: Student Grade Book and System CLI (After Module 4)
3. **Milestone 3**: ATM / Banking System Console Simulation (After Module 5)
4. **Milestone 4**: Library Book Log & Checkout (After Module 6)
5. **Milestone 5**: Monthly Category Expense Tracker (After Module 7)
6. **Milestone 6**: Persistent Todo checklist saving tasks to file (After Module 8)
7. **Capstone**: Complete Command Line Management System (integrating all modules)
